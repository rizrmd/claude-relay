This is a test page to verify the authentication UI implementation

Authentication Flow

The WebSocket server implements the following auth flow:

  1. Client connects via WebSocket
  2. Client sends /auth-status command
  3. Server responds with auth status and URL if needed
  4. User completes authentication in browser
  5. Client sends /auth-token:SESSION_TOKEN
  6. Server validates and enables chat
